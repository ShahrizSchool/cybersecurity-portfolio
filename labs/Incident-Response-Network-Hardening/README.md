# Security Risk Assessment & Network Hardening

## Objective
Identify key vulnerabilities in an organization's network and recommend security hardening methods to prevent future data breaches.

## Scenario
A social media organization experienced a major data breach that exposed customers' personal information. After reviewing the environment, several security weaknesses were identified, including password sharing, default administrative credentials, missing firewall rules, and lack of multifactor authentication.

## Vulnerabilities Identified
- Employees sharing passwords
- Default admin database password
- No firewall traffic filtering rules
- Multifactor authentication (MFA) not implemented

## Hardening Methods Selected
The following security controls were recommended:
- Strong password policies
- Firewall maintenance
- Multifactor authentication (MFA)

## Mitigation Strategy
Recommended security improvements include:
- Enforcing strong password complexity requirements
- Locking accounts after failed login attempts
- Regular firewall rule reviews and updates
- Implementing MFA for all administrative access
